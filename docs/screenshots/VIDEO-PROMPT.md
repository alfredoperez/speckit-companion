# Demo video — product feature showcase

A short **product feature video** for the top of the README, built from **three
real screenshots**. Generated with AI. Keep the screenshots **sharp and legible**
— the point is showing the actual product.

## The three screenshots

- `viewer.png` — reading / managing a spec in the viewer.
- `comments.png` — inline review comments on a spec.
- `activity.png` — the activity overview (phases, tasks, decisions).

## Brand style

- Near-black **#0F0F13** background with a faint blueprint grid (**#2A2A3A**).
- Screenshots framed and floating: rounded corners, a soft blue (**#60A5FA**)
  glow edge, gentle shadow.
- Restrained blues **#60A5FA** / **#3B82F6** with a single yellow **#FACC15**
  highlight accent. **No purple.**
- Any titles in a clean geometric sans; emphasis words in a sketchy hand-drawn
  marker with a yellow highlight. Ink **#E8E8F0**.
- Calm, editorial, premium — gentle motion, no fast cuts, no distortion of the UI.

## Prompt

> A short, premium product feature video for a VS Code extension called SpecKit
> Companion, using the three provided UI screenshots. Show the product clearly:
> reviewing specs inline, tracking progress in an activity panel, and reading a
> spec in the viewer. Present each screenshot framed and floating on a near-black
> #0F0F13 background with a faint blueprint grid, soft blue #60A5FA glow edges, and
> gentle shadows. Restrained blue palette with a single yellow #FACC15 highlight
> accent — no purple. Add brief emphasis words in a sketchy hand-drawn marker with a
> yellow highlight; any titles in a clean geometric sans. Calm, editorial, premium
> motion — gentle, no fast cuts. Keep all screenshots sharp and unaltered.

Arrange and pace it however reads best — this is a loose brief, not a fixed
storyboard.
