03 · Ticket

Implementation-ready boundary

This remains a Living Spec Viewer enhancement. It does not introduce a new product surface or authoring format.

Living Spec Viewer — semantic document components

Outcome: recognized living-spec structures render through accessible Preact components. Unknown Markdown uses the current renderer as a lossless fallback.

Acceptance criteria

  1. Purpose, requirement sections, scenarios, and Uncovered evidence render as semantic components only in living-spec mode.
  2. Requirement cards preserve original document order and exact authored wording.
  3. Acceptance scenarios separate conditions and outcomes without changing their sequence or content.
  4. Uncovered evidence starts with a count and scope statement, then groups omissions by reason in keyboard-accessible disclosures.
  5. Missing confidence, coverage, source, or counts are omitted—never guessed or shown as zero.
  6. Every component preserves source-line identity and existing comment add, restore, edit, and delete behavior.
  7. Dark, light, high-contrast, narrow, and reduced-motion states remain supported through viewer tokens.
  8. Feature specs and unrecognized Markdown remain unchanged; component failure falls back to existing Markdown rendering.
