## Directory Purposes

**`src/`:**
- Purpose: All application source code
- Contains: TypeScript/React components, state management, utilities, tests
- Key files: `main.tsx` (entry), `App.tsx` (root), `types.ts` (types)

**`src/components/`:**
- Purpose: Reusable UI components
- Contains: Presentational components that render UI and accept props/callbacks
- Key files: Header (nav), AddTodo (input form), TodoList (list container), TodoItem (row)

**`src/pages/`:**
- Purpose: Route-mapped page components
- Contains: Components tied to React Router routes (/ and /about)
- Key files: TodosPage, AboutPage

**`src/store/`:**
- Purpose: State management via Context + useReducer
- Contains: Provider component, reducer function, context definition, custom hook
- Key files: `todos.tsx` (logic), `todos.test.ts` (unit tests)

**`src/lib/`:**
- Purpose: Reusable utility functions and helpers
- Contains: Generic adapters and utilities not tied to a specific feature
- Key files: `storage.ts` (localStorage wrapper), `storage.test.ts` (tests)

**`.planning/`:**
- Purpose: GSD artifacts generated by `/gsd-*` commands
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Committed: No — auto-generated, do not edit by hand
- Workflow note: This is brownfield; run `/gsd-map-codebase` once before first phase

**`specs/`:**
- Purpose: Feature specifications for GSD workflow
- Contains: Phase-based specs with spec.md, plan.md, tasks.md per feature
- Committed: Demo fixtures only (`_00_demo-*`); feature specs are gitignored

**`.specify/`:**
- Purpose: Spec-kit CLI configuration and cached state
- Contains: Feature definitions, extension config, integration state
- Committed: Yes, baseline configuration

## Key File Locations

**Entry Points:**
- `index.html`: HTML entry point, loads `/src/main.tsx` as module script
- `src/main.tsx`: React bootstrap, creates root and renders App with BrowserRouter
- `src/App.tsx`: Root component, sets up routing (/ and /about) and TodosProvider

**State & Store:**
- `src/store/todos.tsx`: TodosProvider (context + reducer), useTodos hook
- `src/store/todos.test.ts`: Unit tests for todosReducer (add, toggle, delete, clearCompleted)

**Persistence:**
- `src/lib/storage.ts`: Generic load<T>/save<T> functions (localStorage adapter)
- `src/lib/storage.test.ts`: Tests for round-trip serialization and fallback behavior

**Types:**
- `src/types.ts`: Todo interface (id, text, completed, createdAt)

**Pages:**
- `src/pages/TodosPage.tsx`: Main todo list view, renders AddTodo + TodoList + Clear Completed button
- `src/pages/AboutPage.tsx`: Static about page

**Components:**
- `src/components/Header.tsx`: Navigation bar with links to / and /about
- `src/components/AddTodo.tsx`: Form component for adding new todos (local useState for input)
- `src/components/TodoList.tsx`: Container that renders TodoItem list or empty state
- `src/components/TodoItem.tsx`: Single todo row with checkbox and delete button

**Build Config:**
- `package.json`: Dependencies (React 18, React Router, Vite, vitest, testing-library)
- `vite.config.ts`: Vite setup with React plugin
- `vitest.config.ts`: Test runner with jsdom environment, global test utils, setup file
- `vitest.setup.ts`: Global shims (crypto.randomUUID for jsdom) and cleanup (localStorage.clear after each test)
- `tsconfig.json`: TypeScript strict mode (noUnusedLocals, noUnusedParameters, noFallthroughCasesInSwitch)

**Integration Tests:**
- `src/App.test.tsx`: Full-stack tests for routing, persistence across remounts, UI interactions

## Naming Conventions

**Files:**
- Components: PascalCase with `.tsx` extension (e.g., `Header.tsx`, `AddTodo.tsx`)
- Utilities: camelCase with `.ts` extension (e.g., `storage.ts`)
- Tests: co-located with source using `.test.ts` or `.test.tsx` suffix
- Index files: No barrel exports used; direct imports from file locations

**Directories:**
- Feature dirs: lowercase plural (e.g., `components/`, `pages/`, `store/`, `lib/`)

**Functions & Variables:**
- Components: PascalCase (e.g., `TodosProvider`, `useTodos`)
- Hooks: Prefix with `use` + PascalCase (e.g., `useTodos`)
- Actions: camelCase verbs (e.g., `addTodo`, `toggleTodo`, `clearCompleted`)
- Variables: camelCase (e.g., `todos`, `completed`, `hasCompleted`)
- Context keys: camelCase (e.g., `STORAGE_KEY = 'todos'`)

**Types:**
- Interfaces: PascalCase (e.g., `Todo`, `TodosContextValue`, `AddTodoProps`)
- Discriminated unions: `type Action = { type: '...' } | { type: '...' }`
- Props interfaces: `{ComponentName}Props` suffix (e.g., `TodoItemProps`)

## Where to Add New Code

**New Feature (e.g., tags, priorities):**
- Type definition: Add to `src/types.ts` (extend Todo interface)
- State logic: Add action type and case to `todosReducer` in `src/store/todos.tsx`
- Context methods: Add to `TodosContextValue` interface and TodosProvider implementation
- Tests: Add to `src/store/todos.test.ts` for reducer, `src/App.test.tsx` for integration
- UI component: Create new component in `src/components/{feature}.tsx`
- Integration: Use `useTodos()` hook in page or component to access new state

**New Page/Route:**
- Create route component in `src/pages/{RouteName}.tsx`
- Add `<Route path="/route-name" element={<RouteName />} />` to App.tsx
- Add navigation link to `src/components/Header.tsx`
- Add integration tests to `src/App.test.tsx`

**New Utility:**
- Create `.ts` file in `src/lib/` (e.g., `src/lib/format.ts`)
- Add corresponding `.test.ts` file (e.g., `src/lib/format.test.ts`)
- Import from feature code where needed

**New Component:**
- Create `.tsx` file in `src/components/` with PascalCase name
- Define props interface `{ComponentName}Props`
- Add to `src/App.test.tsx` if testing via route or add isolated `{ComponentName}.test.tsx` if complex
- Import and use in pages or other components

## Special Directories

**`node_modules/`:**
- Purpose: Installed dependencies
- Generated: Yes (via `npm install`)
- Committed: No (gitignored)

**`.planning/`:**
- Purpose: GSD workflow artifacts (codebase maps, phase plans)
- Generated: Yes (by `/gsd-map-codebase`, `/gsd-plan-phase`, etc.)
- Committed: No (auto-generated, do not edit)
- Workflow: Created by GSD commands; consumed by other GSD commands for planning/execution guidance

**`specs/`:**
- Purpose: Feature specifications with phase-based breakdown (spec.md, plan.md, tasks.md)
- Generated: Yes (by GSD or manual creation via spec workflow)
- Committed: Demo fixtures only (_00_demo-specified, etc.); feature specs gitignored
- Workflow Note: One spec per feature; specs author requirements + track progress through spec/plan/tasks/completed states

**`.specify/`:**
- Purpose: Spec-kit CLI local configuration and state
- Generated: Partially (feature.json, integration.json created during setup)
- Committed: Yes, baseline configuration
- Workflow: Checked in so team shares spec-kit settings

**`.vscode/`:**
- Purpose: VS Code workspace settings
- Generated: No (manually configured)
- Committed: Yes (team editor config)
