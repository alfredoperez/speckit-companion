<objective>
Add a 3-tier priority (low/medium/high) to each todo: new todos default to medium, every row shows a colored priority badge, and the list stays sorted high-to-low — without breaking any existing todo behavior. Delivers PRIO-01 through PRIO-05 as a single vertical slice.

Purpose: Give users a way to organize todos by importance so the most important work surfaces first.
Output: A Priority type + Todo.priority field, store default + sort logic, a PriorityBadge component, a priority selector in the add form, and updated store/component/integration tests.
</objective>

<phase_goal>
## Phase Goal (user story)

**As a** person managing a todo list, **I want to** set and see each todo's priority and have the list ordered by it, **so that** I can focus on my highest-priority work first.
</phase_goal>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user -> React state | Todo text and priority selection enter app state from the UI |
| React state -> localStorage | Todos (including priority) are persisted and re-read across reloads |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | localStorage `priority` field | medium | mitigate | `sortByPriority` and `PriorityBadge` default any missing/unknown priority to medium (`?? 'medium'` / `?? map.medium`), so a hand-edited or legacy value cannot break rendering or ordering |
| T-01-02 | Tampering (XSS) | todo text rendered in list | low | accept | React escapes text content by default; this phase adds no `dangerouslySetInnerHTML` and renders priority only as a constrained union value |
| T-01-03 | Information Disclosure | priority stored in plaintext localStorage | low | accept | Non-sensitive UI preference; app is localStorage-only by design (no backend) |
| T-01-SC | Tampering | package installs | n/a | accept | No new dependencies added this phase — nothing to install, no supply-chain surface |
</threat_model>
