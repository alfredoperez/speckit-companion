# Implementation Plan: Clarify AI Provider Dropdown Labels

**Branch**: `108-clarify-provider-labels` | **Date**: 2026-05-26 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/108-clarify-provider-labels/spec.md`

## Summary

The AI provider dropdown exposes raw internal enum keys (`ide-chat`, `claude-vscode`, `copilot`) instead of the brand names users recognize. The fix has three layers: (1) add `enumItemLabels` to `package.json` so the VS Code settings editor shows friendly static names; (2) update all `displayName` strings in `PROVIDER_PATHS` to use full product names; (3) extract the existing IDE detection logic from `IdeChatProvider` into a standalone `getProviderDisplayName(type)` function so every UI surface (QuickPick, sidebar tree) resolves the `ide-chat` label dynamically at render time.

## Technical Context

**Language/Version**: TypeScript 5.3+ (ES2022, strict mode)  
**Primary Dependencies**: VS Code Extension API (`@types/vscode ^1.84.0`); `vscode.env.uriScheme` / `vscode.env.appName` for IDE detection  
**Storage**: Static ‚Äî `package.json` `enumItemLabels`; runtime strings from `vscode.env`

## Approach

### Change 1 ‚Äî `package.json`: Add `enumItemLabels`

Add the `enumItemLabels` array alongside the existing `enum` and `enumDescriptions` arrays in `speckit.aiProvider`:

```json
"enumItemLabels": [
  "Claude Code",
  "Claude Code (VS Code)",
  "Gemini CLI",
  "GitHub Copilot CLI",
  "Codex CLI",
  "Qwen Code",
  "OpenCode",
  "IDE Chat (Copilot ¬∑ Cursor ¬∑ Windsurf)"
]
```

Order must match the `enum` array exactly: `claude`, `claude-vscode`, `gemini`, `copilot`, `codex`, `qwen`, `opencode`, `ide-chat`.

---

### Change 2 ‚Äî `aiProvider.ts`: Update `displayName` strings

Update every `displayName` in `PROVIDER_PATHS` to the full product name:

| Key             | Old                   | New                                                            |
| --------------- | --------------------- | -------------------------------------------------------------- |
| `CLAUDE`        | `'Claude'`            | `'Claude Code'`                                                |
| `CLAUDE_VSCODE` | `'Claude in VS Code'` | `'Claude Code (VS Code)'`                                      |
| `GEMINI`        | `'Gemini'`            | `'Gemini CLI'`                                                 |
| `COPILOT`       | `'Copilot'`           | `'GitHub Copilot CLI'`                                         |
| `CODEX`         | `'Codex'`             | `'Codex CLI'`                                                  |
| `QWEN`          | `'Qwen'`              | `'Qwen Code'`                                                  |
| `OPENCODE`      | `'OpenCode'`          | `'OpenCode'` _(unchanged)_                                     |
| `IDE_CHAT`      | `'IDE Chat'`          | `'IDE Chat'` _(static fallback; runtime replaced by Change 3)_ |

Also add this export after `PROVIDER_PATHS`:

```ts
/**
 * Returns the human-readable display name for the given provider type.
 * For IDE_CHAT, resolves the host editor at call-time and returns
 * "GitHub Copilot", "Cursor Chat", "Windsurf Chat", or "IDE Chat" as fallback.
 * All other providers return their static PROVIDER_PATHS displayName.
 */
export function getProviderDisplayName(type: AIProviderType): string {
  if (type === AIProviders.IDE_CHAT) {
    return getIdeChatDisplayName();
  }
  return PROVIDER_PATHS[type].displayName;
}
```

### Change 3 ‚Äî `ideChatProvider.ts`: Export `getIdeChatDisplayName`

Extract the detection-to-label mapping into a standalone exported function (no class needed):

```ts
const IDE_DISPLAY_NAMES: Record<HostIde, string> = {
  vscode: "GitHub Copilot",
  cursor: "Cursor Chat",
  windsurf: "Windsurf Chat",
  antigravity: "IDE Chat",
  unknown: "IDE Chat",
};

export function getIdeChatDisplayName(): string {
  const scheme = (vscode.env.uriScheme || "").toLowerCase();
  const appName = (vscode.env.appName || "").toLowerCase();
  let host: HostIde = "unknown";
  if (scheme === "cursor" || appName.includes("cursor")) host = "cursor";
  else if (scheme === "windsurf" || appName.includes("windsurf"))
    host = "windsurf";
  else if (
    scheme === "antigravity" ||
    scheme === "agy" ||
    appName.includes("antigravity")
  )
    host = "antigravity";
  else if (
    scheme === "vscode" ||
    scheme === "vscode-insiders" ||
    appName.includes("visual studio code")
  )
    host = "vscode";
  return IDE_DISPLAY_NAMES[host];
}
```

`IdeChatProvider.detectHostIde()` can then delegate to this function to avoid duplication.

### Change 4 ‚Äî `steeringExplorerProvider.ts`: Use `getProviderDisplayName`

The provider tree header currently sets its label to `providerPaths.displayName` (static string). Replace:

```ts
// Before
items.push(new SteeringItem(
    providerPaths.displayName,
    ...
));

// After
items.push(new SteeringItem(
    getProviderDisplayName(providerType),
    ...
));
```

Import `getProviderDisplayName` from `aiProvider.ts`.

### Not changing

- The `quickPickDescription` strings in `PROVIDER_PATHS` ‚Äî they already describe each provider clearly.
- The `IAIProvider` interface or `name` property on provider classes ‚Äî `IdeChatProvider.name = 'IDE Chat'` can stay as the stable internal name; the display label is separate.
- Any test file that asserts on `displayName` strings ‚Äî those will need updating in the tasks step.
