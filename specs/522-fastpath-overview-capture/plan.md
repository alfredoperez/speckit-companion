# Implementation Plan: Fast path fills the Overview

## Summary

Close the two Overview gaps on a fast-tracked spec by capturing more into `.spec-context.json`. First, add a `write-context.py --set approach=…` call to the `specify` fast-path finalize node so the Overview's APPROACH card (which reads `ctx.approach`) is populated. Second, add a small deterministic recorder — `speckit-extension/scripts/record-living-specs.py` — that reads the living-specs registry, gates on `enabled: true`, runs the existing `resolve-spec-paths.py` resolver over the changed files, and records the matched capabilities onto `livingSpecs.loaded` via the existing `capture.set_living_specs_loaded` path. The `load-living-specs` and fast-path `finalize` node bodies call this one line instead of asking the AI to gate-and-decide, so the recording becomes reliable while the AI's *reading* of specs for drafting stays best-effort prose. Everything stays best-effort/opt-in/read-only: any miss is a silent no-op that exits 0.
