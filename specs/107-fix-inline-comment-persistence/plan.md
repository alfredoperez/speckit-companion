# Implementation Plan: Fix Inline Comment Persistence

**Branch**: `107-fix-inline-comment-persistence` | **Date**: 2026-05-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/107-fix-inline-comment-persistence/spec.md`

## Summary

Inline review comments added to `spec.md` are silently dropped ‚Äî they never reach
`.spec-context.json` and never restore on reopen. Comments on `plan.md` and `tasks.md` work
correctly. Root cause (confirmed via code trace): `currentDoc()` in the webview returns `null` when
`spec.md` is active because `navState.currentDoc` equals `'specify'` (the workflow step name), not
`'spec'` (the `CoreDocumentType`). The fix normalises `'specify'` ‚Üí `'spec'` inside `currentDoc()`.
A secondary improvement tightens `sourceDoc` lookups in the extension's message handlers.

## Approach

### Fix 1 ‚Äî `webview/src/spec-viewer/editor/currentDoc.ts`

**Before**:
```ts
const d = navState.value?.currentDoc;
return d === 'spec' || d === 'plan' || d === 'tasks' ? d : null;
```

**After**:
```ts
const d = navState.value?.currentDoc;
if (d === 'spec' || d === 'specify') return 'spec';
if (d === 'plan' || d === 'tasks') return d;
return null;
```

This single change unblocks all three consumers ‚Äî `persistAdd`, `restoreComments`, and
`submitAllRefinements` ‚Äî because they all call `currentDoc()` as their gate.

---

### Fix 2 ‚Äî `src/features/spec-viewer/messageHandlers.ts` (2 locations)

`handleAddComment` and `dispatchDocRefinement` both do:
```ts
const sourceDoc = instance.state.availableDocuments.find(d => d.isCore && d.type === doc);
```
When `doc = 'spec'` and `spec.md` has `type = 'specify'` in `availableDocuments`, this returns
`undefined`.

**After** (both sites):
```ts
const sourceDoc = instance.state.availableDocuments.find(
    d => d.isCore && (d.type === doc || d.fileName === `${doc}.md`)
);
```

`handleAddComment`: `sourceLines` will now contain the actual file lines ‚Üí richer anchor context
stored with each comment.

`dispatchDocRefinement`: `filename` will now be `sourceDoc.fileName` (`'spec.md'`) rather than
the `${doc}.md` fallback ‚Äî explicit rather than coincidental.
