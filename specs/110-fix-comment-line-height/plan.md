# Implementation Plan: Fix Comment Line Height

**Branch**: `110-fix-comment-line-height` | **Date**: 2026-05-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/110-fix-comment-line-height/spec.md`

## Summary

Inline review comments in the spec viewer cause their host lines to grow taller
than equivalent uncommented lines, and every task item has a persistent empty-row
gap due to the `.line-comment-slot` flex child. Fix is CSS-only: suppress the
empty slot via `:empty { display: none }` and remove the task-item slot's
redundant `margin-top`. Add four Storybook stories to `InlineComment.stories.tsx`
so the height parity is visually verifiable without running the full extension.
