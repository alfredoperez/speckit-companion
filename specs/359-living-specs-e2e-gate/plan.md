# Plan — Living Specs end-to-end validation gate (LS·4)

**Issue:** #364 · **Spec:** `specs/359-living-specs-e2e-gate/spec.md`

## Problem

Living Specs production logic (resolver, auto-load, fold-back) is merged across LS·1–3, but nothing proves the loop runs end-to-end. v1's "done" gate is a single test that drives two features through a real fold against one disposable repo and confirms the living spec accumulates both — plus an opt-out run proving zero side effects when the feature is off.

## Approach

This is a harness-only change. The fold logic, resolver, and check script already exist and are exercised by LS·3. LS·4 layers a thin runner on top:

1. **Accumulation arrange** — bake one repo (extending the LS·3 fixture shape) with a `todos` capability + a real living spec, a feature A whose spec carries **no** delta block (additive), and a feature B whose spec carries a real `## ADDED Requirements` delta. Both feature dirs get a finished `.spec-context.json` (the state mark-complete folds from) and the repo gets a feature branch with an in-scope change so the fold's merge-base diff yields `src/todos/**`.
2. **Accumulation act** — read the living spec (before) → `runFold(A)` → read (after-A) → `runFold(B)` → read (after-B). All via the existing real `runFold` / `readText`.
3. **Accumulation assert** — after-A == before (no-op); after-B contains B's requirement; after-B still contains the original requirement (no clobber); monotonic growth. Capture real unified diffs (before→after-A is empty; before→after-B shows B's block).
4. **Opt-out arrange/act/assert** — bake a sibling repo with `enabled: false`, snapshot the `capabilities/**` tree + capability spec bytes, `runFold` the delta feature, re-snapshot. Assert byte-equality and an empty file-tree diff (no new `capabilities/**` files), and no `livingSpecs.synced`.
5. **Evidence** — assemble `LS4.json` with both runs, honest `mode` per run (`real+seeded-spec` accumulation, `deterministic` opt-out), assertions, verdict. Append the trend row. Reuse the existing `runPytest` for the green-suite assertion.
6. **Docs/vault** — append an LS·4 section to the vault status page, flip the LS·4 row to shipped, bump the shipped count to 4/8.

## Summary

**Problem:** Living Specs has shipped piece by piece but never been proven to work as a whole loop. **Solution:** a single end-to-end harness run that brings an area under a living spec, drives two features through it (one that should change nothing, one that should fold a new requirement in), and confirms the living spec ends up holding both with nothing lost — plus a switched-off run that proves it leaves everything alone. The result is recorded as honest, repo-relative evidence and reflected on the status page, which marks Living Specs v1 done.
